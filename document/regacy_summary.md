1. 현재 핵심 로직 요약 (Current Logic Summary)
데이터 수집 및 전처리 (Data Fetching & Processing)

외부 API 연동: KisClient 클래스를 통해 한국투자증권(KIS) API와 통신하여 주식 현재가, 기간별 차트 데이터(일봉 등), 투자자별 매매동향(외국인/기관 순매수) 데이터를 JSON 형태로 수집합니다.

데이터 병합 및 지표 계산: DataFetcher가 KIS API에서 가져온 차트 데이터와 수급 데이터를 Pandas DataFrame으로 병합하고 숫자형 변환 및 결측치를 처리합니다. 이후 calculate_indicators 함수를 통해 MA(이동평균선)와 MACD 등의 보조지표를 계산합니다.

핵심 비즈니스 로직 (Strategy Analysis)

다중 종목 스캔: MarketScanner가 로컬 tickers.json에서 시가총액 상위 종목을 불러온 뒤, ThreadPoolExecutor를 이용해 멀티스레딩(최대 10개)으로 빠르게 다수 종목의 데이터를 수집하고 분석합니다.

주도주 판별 전략 (StockStrategy):

P1 (지수 주도주): 시가총액과 당일 등락률을 곱하여 시장 지수 기여도 점수를 산출합니다.

P2 (수급 주도주): 가격(1~5% 등락), 거래대금(150억~500억 이상), 이격도(5일선 기준 5% 이내) 필터를 거친 후, 외국인/기관의 3일 연속 순매수 또는 쌍끌이 조건을 만족하는지 검사합니다.

Sniper (급등 전조): 전일 대비 거래량이 50% 이상 급증하면서 주가가 0~5% 내로 눌려있는 종목을 포착합니다.

UI 렌더링 (Frontend)

Streamlit 기반의 app.py에서 사용자로부터 시장(KOSPI/KOSDAQ)과 종목 수를 입력받아 스캔을 트리거합니다.

분석이 완료된 DataFrame을 P1, P2, Sniper 세그먼트로 나누어 화면에 테이블 형태로 렌더링하며, 금액 단위를 '억', '원' 등으로 변환하는 포맷팅을 수행합니다.

2. 페인포인트(문제점) 진단 (Pain Points & Structural Flaws)
현재 코드는 작동은 하지만, 소프트웨어 공학(객체지향 설계 및 유지보수성) 관점에서 다음과 같은 심각한 구조적 결함(기술 부채)을 가지고 있습니다.

① 강한 결합도와 UI/로직의 혼재 (MVC 패턴 부재)

문제: 프론트엔드인 app.py가 화면만 그리는 것이 아니라, DataFetcher와 StockStrategy를 직접 임포트하여 사이드바의 '종목 정밀 진단' 로직을 직접 실행하고 있습니다. 또한 금액을 '억' 단위로 변환하는 등의 데이터 가공 로직이 뷰(UI) 렌더링 코드에 하드코딩되어 섞여 있습니다.

영향: UI 코드가 너무 비대해지고, 분석 로직이 변경되면 UI 코드까지 함께 수정해야 하는 사이드 이펙트가 발생합니다.

② 개방-폐쇄 원칙(OCP) 위배 및 확장성 한계

문제: StockStrategy 클래스 내부의 analyze 메서드가 P1, P2, Sniper 로직을 절차지향적으로 순차 실행한 뒤, 문자열 포맷팅(reasons.append(...))과 우선순위(priority)를 한 곳에서 범벅으로 처리하고 있습니다.

영향: 새로운 전략(예: P3)을 추가하거나 기존 지표를 변경하려면, 기존 코드를 뜯어고쳐야 하므로 예기치 않은 버그가 발생할 위험이 높습니다. 알고리즘들이 서로 분리되지 않아 '전략 패턴(Strategy Pattern)'의 적용이 시급합니다.

③ 단일 책임 원칙(SRP) 위배

문제: MarketScanner 클래스가 종목 리스트 로드(JSON 파싱), 멀티스레딩 제어, 데이터 패치 호출, 에러 핸들링 등 너무 많은 책임을 동시에 수행하고 있습니다.

영향: 스캔 프로세스 중 한 부분에서 문제가 발생했을 때 원인을 추적하거나 단위 테스트(Unit Test)를 작성하기가 매우 어렵습니다.

④ 예외 처리의 파편화 및 의존성 역전(DIP) 부재

문제: 에러가 발생하면 표준 예외(Exception)를 던지지 않고 튜플 형태(df, error)나 딕셔너리({'error': ...})로 문자열을 반환하여 릴레이식으로 전달하고 있습니다. 또한, 하위 모듈이 외부 API 모듈(KisClient)을 직접 생성하고 의존하여 추상화가 되어있지 않습니다.
