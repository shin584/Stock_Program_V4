[Use Case 명세서] 시장 주도 섹터 및 종목 스캔 (Stock_Program_V4) - V2
1. 개요 (Context)
목적: 주식 시장의 주도 종목(P1, P2, P3)을 식별하고, 이를 바탕으로 시장 컨텍스트를 대변하는 상위 3개 주도 섹터를 도출하여 트레이딩 전략 수립을 지원한다.

주 액터 (Primary Actor): 개인 투자자

부 액터 (Secondary Actor): KIS API (한국투자증권 데이터 제공 시스템)

사전 조건: [개인 투자자]는 시스템에 접속된 상태이며, 시스템 내부적으로 KIS API 인증 토큰이 유효하게 관리되고 있어야 한다.

2. 메인 시나리오 (Main Success Scenario)

지시: 아래 흐름은 향후 시스템 아키텍처의 제어 흐름(Sequence) 설계 기준으로 사용됨.

[개인 투자자]가 [대시보드 UI]에서 분석 기준일을 선택한다.

[개인 투자자]가 [대시보드 UI]의 '종목 스캔' 버튼을 클릭한다.

[대시보드 UI]가 [컨트롤러]로 스캔 시작 및 분석을 요청한다.

[컨트롤러]가 [KIS API]를 호출하여 대상 종목들의 시세 및 투자자 매매동향 데이터를 요청한다.

[KIS API]가 [데이터 처리 모듈]로 Raw JSON 데이터를 반환한다.

[데이터 처리 모듈]이 반환된 데이터를 정형화(DataFrame)하고 필수 보조지표를 계산한다.

[전략 분석 모듈]이 전처리된 데이터를 바탕으로 각 종목의 P1, P2, P3 조건 부합 여부를 검사한다.

[컨트롤러]가 분석 완료된 종목 리스트를 바탕으로 상위 3개 주도 섹터를 집계한다.

[대시보드 UI]가 [개인 투자자]에게 필터링된 종목 리스트와 주도 섹터 카드를 시각화하여 렌더링한다.

3. 대안 및 예외 흐름 (Alternative/Exception Flows)
3a. KIS API 토큰 만료 시: 

[KIS API]가 [컨트롤러]로 인증 실패(토큰 만료) 에러를 반환한다.

[컨트롤러]가 [인증 관리 객체(Singleton)]로 새로운 토큰 발급을 요청하여 갱신한다.

[컨트롤러]가 중단된 데이터 요청을 [KIS API]로 재시도한다.

3b. 초당 호출 횟수 초과 (Rate Limit) 시: 

[KIS API]가 [컨트롤러]로 호출 제한 초과 에러 메시지를 반환한다.

[컨트롤러]가 지수 백오프(Exponential Backoff) 로직에 따라 일정 시간 대기한 후, [KIS API]로 해당 종목의 요청을 재시도한다.

3c. 분석 기준일이 주식 시장 휴장일인 경우:

[KIS API]가 [컨트롤러]로 해당 날짜의 시세 데이터가 없음을 알리는 응답을 반환한다.

[컨트롤러]가 해당 날짜를 '시장 휴장일'로 판단하고 전체 스캔 프로세스를 중단한다.

[대시보드 UI]가 [개인 투자자]에게 "선택하신 날짜는 시장 휴장일입니다. 다른 날짜를 선택해주세요."라는 에러 메시지를 표시한다.

3d. 특정 종목의 데이터가 부족하여 분석에 실패한 경우:

[데이터 처리 모듈] 혹은 [전략 분석 모듈]이 특정 종목에 대해 '데이터 부족' 상태를 감지한다.

[컨트롤러]가 해당 종목의 코드와 이름을 '분석 실패 리스트(Failures)'에 기록한다.

[컨트롤러]가 중단 없이 다음 종목의 데이터 요청 및 분석을 계속 진행한다.

[대시보드 UI]가 스캔 결과 화면 하단에 [분석 실패 종목 보기] 섹션을 생성하고, 실패한 종목의 이름과 구체적인 사유(예: "데이터 부족", "신규 상장주 등")를 테이블 형태로 나열하여 [개인 투자자]에게 보여준다.

4. 비즈니스 룰 (Business Rules)
지시: 아래 규칙은 전략 패턴(Strategy Pattern) 기반의 단위 테스트 및 순수 함수 작성 시 '채점 기준'으로 엄격하게 적용됨.

Rule 1. P1 (지수 주도주) 판별 로직

지수 기여도 점수: 시가총액 × 당일 등락률.

조건: 산출된 지수 기여도 점수가 0을 초과하면 P1 종목으로 식별한다.

Rule 2. P2 (수급 주도주) 판별 로직

가격 (Price): 당일 등락률이 1.0% 이상 5.0% 이하여야 한다 (음봉 허용).

유동성 (Liquidity): 거래대금이 150억 원 이상이어야 한다. 단, 등락률이 2.0% 미만인 경우 모멘텀 보완을 위해 거래대금 500억 원 이상을 요구한다.

이격도 (Disparity): 이동평균선이 정배열(MA5 > MA20) 상태일 때, 종가 기준 5일선 이격도가 5.0% 이하여야 한다 (과열 상태 진입 방지).

수급 (Trigger): 외국인 3일 연속 매수, 혹은 기관 3일 연속 매수, 혹은 쌍끌이 매수(외인/기관 각각 최소 1일 이상 & 합산 2일 이상) 조건 중 하나를 만족해야 한다.

규모 (Magnitude): 최근 5일 누적 기관/외인 순매수액이 100억 원 이상이거나, 당일 기관/외인 순매수액이 30억 원 이상이어야 한다.

Rule 3. P3 (Sniper / 급등 전조) 판별 로직

가격 (Price): 당일 등락률이 0.0% 초과 5.0% 이하여야 한다 (주가 눌림).

거래량 (Volume Ratio): 당일 거래량이 전일 대비 50% 이상 급증해야 한다.

Rule 4. 주도 섹터 집계 (Sector Aggregation) 로직:

스캔을 통과한 P1, P2 종목들을 산업 섹터별로 맵핑한다.

각 섹터에 속한 P1, P2 종목의 개수(Count)를 합산하여 가장 많은 종목을 배출한 상위 3개 섹터를 '오늘의 주도 섹터'로 도출한다.
